# Contributing to Flutter State Migrator

First off, thank you for considering contributing to the Flutter State Migrator! It's people like you that make the Flutter community great.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please open an issue and include:
- A clear description of the issue.
- Steps to reproduce.
- A snippet of the Dart code causing the issue.
- The output of the migrator with the `--dry-run` flag.

### Suggesting Enhancements
We are always looking to support more state management libraries and patterns. Open an issue with the "enhancement" label to discuss your idea.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`dart test`).
4. Format your code (`dart format .`).
5. Open a PR with a clear description of the changes.

## Development Setup

1. Clone the repository.
2. Run `dart pub get`.
3. Use `dart run bin/migrator.dart` to test your changes locally.
